# Phase 3 — Thermal & Contact

Implements baseline thermal/contact/model-change physics plus the one new capability
(`contact-search`). Each task cites the capability it satisfies. Depends on
`phase-1-foundation` and `phase-2-nonlinear-statics-and-materials`.

## 1. Contact-search engine (spec: contact-search [NEW])

- [x] 1.1 Spatial proximity search (bounding structure) with configurable search/adjust distance — a UNIFORM GRID over the master faces (`fem::MasterGrid`, include/calculixpp/fem/contact_search.hpp + src/fem/contact_search.cpp). Each master face (`MasterFace{elem_index, face}`, resolved from a *SURFACE by `expand_master_faces`) has its axis-aligned bounding box (`face_aabb`) INFLATED by the configurable search/adjust distance and rasterized into the grid cells it overlaps. The cell size auto-sizes to `max(mean padded face extent, search)` so a query point plus its search radius always fits in a 3x3x3 cell neighborhood — `MasterGrid::candidates(p)` gathers only the faces in that neighborhood (deterministic, de-duplicated), a broad-phase superset that NEVER visits every master face. This is the O(1)-per-query bucketing that avoids the O(n_slave * n_master) all-pairs test. VALIDATED (tests/test_contact_search.cpp): on a 40-hex strip a query near element 20 returns only its local neighborhood (candidate count < 40, far faces pruned, out-of-range queries return empty); scaling over n in {20,40,80} keeps the summed candidate work <= 12n (and far below n^2), proving sub-quadratic search. (spec: contact-search — spatial proximity search / large surfaces avoid all-pairs cost.)
- [x] 1.2 Node-to-surface projection: closest point, signed gap, local normal/tangent frame — `fem::project_onto_face` runs a Gauss-Newton closest-point iteration on the master face's natural (parametric) coordinates (residual `r_a = (x_slave - x(xi,eta)) . t_a`, curvature-dropped Jacobian `J_ab = -t_a.t_b`), clamped to the face domain (unit triangle for tet/wedge-end faces, [-1,1]^2 for quad faces) each step, reusing the SHARED face shape/tangent machinery via the new `fem::face_frame_at` (added to fem/loads.{hpp,cpp}). It returns the projected point, its local (xi,eta), the signed normal gap `g = (x_slave - x_proj).n` (g<0 = penetration, g>0 = clearance), and an orthonormal local frame (normal + two tangents). The master NORMAL is oriented OUTWARD (away from the element centroid) so it is topology-robust across the element families (the raw `t_xi x t_eta` sign follows the stored face ordering the pressure kernel consumes). `fem::project_nearest` chains the grid broad phase + per-candidate projection and keeps the closest face within the search distance (invalid when none). VALIDATED ANALYTICALLY (tests/test_contact_search.cpp): projecting a node onto a KNOWN planar hex face gives the exact footprint point, outward +z normal, and correct gap SIGN (+0.25 above, -0.1 penetrating); a node exactly on the face has g~0 (~1e-10); an off-face node clamps to the nearest face-boundary point; the nearest-face selection picks the right element. (spec: contact-search — node-to-surface projection.)
- [~] 1.3 Surface-to-surface mortar segment integration (dual + standard Lagrange basis) — DEFERRED (HONEST). Node-to-surface PENALTY contact is the complete, validated core (tasks 1.1-1.6, 1.4/1.5 above): spatial search, closest-point projection, gap/pressure-overclosure, normal + Coulomb-friction operator assembled into the Newton tangent/residual, and the shared gap/pressure/area state that thermal contact (2.4) now consumes. The mortar surface-to-surface formulation (`contactmortar.c` / `multimortar.c`: dual/standard-Lagrange segment integration over overlapping master/slave face segments on non-matching meshes, condensable Lagrange multipliers) is research-level and remains a follow-up — it is NOT faked. `build_contact_pairs` (mechanical) and `build_thermal_contact_pairs` (thermal) both reject a `TYPE=SURFACE TO SURFACE` pair with a clear message naming the deferral, so a mortar deck fails loudly rather than being silently mis-solved as node-to-surface. Recorded in openspec/BACKLOG.md. (spec: contact-search — mortar integration; contact — mortar S2S.)
- [x] 1.4 Gap / pressure-overclosure evaluation (NORMAL + TANGENTIAL) — NORMAL: `fem::contact_force_weight` (src/fem/contact.cpp) evaluates the normal contact restoring force from the signed gap under the active `*SURFACE BEHAVIOR` law: HARD/LINEAR give the linear penalty `F = kappa*(-g)` for `g<0` (zero for clearance), EXPONENTIAL a bounded-slope softened `F = kappa*c0*(exp(-g/c0)-1)`. The pressure-overclosure law is supplied by the surface behavior (`Model::contact_behavior`), and the geometric `fem::ContactState.pressure/area` fields (task 1.6) stay the shared thermal-reuse handle. TANGENTIAL (Coulomb friction, with 2.3): under genuine normal pressure (g<0) the operator forms the tangential slip `g_t = (I - n⊗n)(u_slave - Σ N_i u_master_i)` of the slave vs its master footprint, an elastic STICK predictor `t_trial = -stick * g_t`, and return-maps it onto the Coulomb cone `|t| <= mu*p` (`fem::friction_return_map`): inside the cone STICKS (`t = t_trial`), outside SLIPS (`t` capped at `mu*p` along `ĝ_t`). VALIDATED: tests/test_contact.cpp test_overclosure_law (clearance -> 0, `F = k*pen`, touching -> 0) + test_friction_return_map / test_friction_operator_stick_slip (the exact stick force `-stick*g_t` below the cone, the SATURATED `mu*p` traction above it, and the mu-scaling of the cap — all read from the operator's f_int contribution). (spec: contact-search — normal + tangential traction from the interface laws.)
- [x] 1.5 Contact operator generation assembled within the Newton iteration — `fem::build_contact_pairs` resolves each `*CONTACT PAIR` into slave nodes + master faces + a `fem::MasterGrid` broad phase + the behavior law + an auto/explicit penalty stiffness; `fem::add_contact` (called inside `numerics::solve_nonlinear_static`'s `assemble_at` each iteration) projects every slave node onto the nearest master face in the DEFORMED configuration (`fem::face_eval_deformed` = reference geometry + current nodal displacement, reusing the shared face shape/tangent machinery), runs an ACTIVE-SET update (penetrating `g<0` -> force; within a small near-contact adjust band -> stiffness only, killing the interface rigid mode; clearance beyond -> released/tensile -> inactive), and adds the normal contact force to the Newton RESIDUAL (`f_int`) and the consistent penalty `kt (dg/du)⊗(dg/du)` (the `k n⊗n` operator across slave↔master DOFs) to the TANGENT through the constraint transform (`scatter_contact_block`, the same congruence path elements use). A `*CONTACT PAIR` deck routes to `solve_nonlinear_static` (CLI + Python auto-dispatch on `Model::has_contact()`); a deck with NO contact never builds any of this and is byte-for-byte the pre-contact path. VALIDATED (tests/test_contact.cpp + python test_contact_two_block_equilibrium_python): the analytical TWO-BLOCK test — two stacked C3D8 cubes meeting only through a `*CONTACT PAIR` (separate elements/nodes on the shared plane), base fixed, top loaded by F — the base reaction balances F exactly (global equilibrium, ~1e-6) with small penetration `~ F/(4*kappa)` that shrinks as the penalty stiffens; end to end the stock-CalculiX `contact1` node-to-surface deck converges with the right compression sign/order (its exact NLGEOM+exponential match is not gated). (spec: contact-search — contact contributes to the tangent and residual via the ComputeBackend.)
- [x] 1.6 Expose gap/pressure/area state for thermal contact reuse — the per-contact `fem::ContactState` struct (contact_search.hpp) carries the full interface geometry (validity, master-face index, local xi/eta, projected point, unit normal + two tangents, signed gap) PLUS `pressure` and `area` fields the geometric core leaves at zero (law-agnostic). It is the single interface-geometry object that mechanical contact (pressure from the overclosure law, task 1.4/1.5) and thermal contact (gap conductance / gap heat generation keyed on gap/pressure/area) will both consume, so the same search feeds both without a second projection. VALIDATED: tests/test_contact_search.cpp test_state_exposes_thermal_fields (the struct exposes the reuse fields, defaulted in the geometric core). (spec: contact-search — shared search for thermal contact.)

## 2. Contact modeling (spec: contact)

- [x] 2.1 `*CONTACT PAIR` (NODE TO SURFACE) and `*SURFACE INTERACTION` — the parser accepts `*CONTACT PAIR, INTERACTION=<name>, TYPE=NODE TO SURFACE` (slave/master `*SURFACE` names on the data line; `TYPE=SURFACE TO SURFACE` parses but the solver rejects it as mortar-deferred) and `*SURFACE INTERACTION, NAME=<name>`, stored on `Model::contact_pairs` / `Model::surface_interactions` (`core/contact.hpp`). `*SURFACE` face ids now cover hex/wedge faces (S1..S6). A `*CONTACT PAIR` deck routes to `solve_nonlinear_static` with the contact contribution. VALIDATED: tests/test_contact.cpp + python (the two-block deck parses and solves end to end through the CLI and `solve`/`solve_text`). Surface-to-surface (mortar) is [~] with 1.3. (spec: contact — contact pairs / surface interaction.)
- [x] 2.2 `*SURFACE BEHAVIOR` (hard / linear / exponential pressure-overclosure) — parsed into `SurfaceBehavior{law, k, p0, c0}` on the named `*SURFACE INTERACTION` (`PRESSURE-OVERCLOSURE=HARD|LINEAR|EXPONENTIAL`; TIED treated as hard). HARD auto-sizes a large penalty `~50*E*h` from the adjacent master element (near-zero overclosure); LINEAR uses the user slope; EXPONENTIAL uses the two-parameter `(p0,c0)` card as a bounded-slope softened law. Consumed by `fem::contact_force_weight` / the operator (tasks 1.4/1.5). VALIDATED: tests/test_contact.cpp (law shape + the two-block penetration `~F/(4*kappa)` scaling with the linear slope). Tied pressure-overclosure as a genuine bilateral constraint is approximated by a stiff hard penalty. (spec: contact — surface behavior normal.)
- [x] 2.3 `*FRICTION` (Coulomb, stick stiffness, stick↔slip) — IMPLEMENTED. The parser reads `*FRICTION` (data line `mu [, stick_stiffness]`) on the `*SURFACE INTERACTION`, stored as `Friction{mu, stick, has}` (core/contact.hpp) and resolved by `Model::contact_friction`. The penalty node-to-surface operator (`fem::add_contact`) adds a tangential Coulomb traction to the SAME Newton residual + tangent as the normal contact, active only under normal pressure (g<0): the elastic STICK predictor `t_trial = -stick * g_t` is return-mapped onto the friction cone `|t| <= mu*p` (`fem::friction_return_map`) — STICK inside (tangential penalty / stick stiffness), SLIP outside (return-mapping onto the cone: traction capped at `mu*p` in the slip direction). The CONSISTENT regime tangent (`fem::friction_tangent_matrix`, scattered by `add_friction_tangent` = `B Dmat Bᵀ` through the constraint transform) is `stick*P` while stuck and the small slip-direction tangent `mu*p/|g_t|(P - ŝ⊗ŝ)` once slipping — the latter is what makes the stick↔slip Newton step converge instead of chattering against the capped force. The stick/slip state is (re)decided from the trial each Newton iteration (active-set within the loop). The auto stick stiffness (when the `*FRICTION` second field is absent) is sized to the master element shear scale (`~0.1*E*h`, capped by the normal penalty) so the stick↔slip transition converges. A frictionless deck (no `*FRICTION`, or mu==0) is byte-for-byte the normal-only contact path. VALIDATED ANALYTICALLY: a block held under normal pressure and dragged tangentially STICKS (traction `-stick*g_t`, below `mu*p`) below the Coulomb limit and SLIPS (traction saturated at `mu*p`) above — read exactly from the operator's interface force in tests/test_contact.cpp (`test_friction_return_map`, `test_friction_operator_stick_slip`: stick force, saturated slip cap, mu-scaling of the cap), plus an end-to-end pressed-together friction solve that converges with the friction contribution in the tangent/residual (`test_friction_end_to_end_converges` + python `test_contact_friction_stick_slip_python`). No clean in-scope stock-CalculiX friction `.dat.ref` is gated (the stock `friction1` n2s stick deck uses NLGEOM + `*CONTACT PRINT CDISP/CSTRESS` output not produced here) — the analytical stick/slip transition validates the physics. (spec: contact — tangential contact / stick-slip.)
- [x] 2.4 Thermal contact: `*GAP CONDUCTANCE`, `*GAP HEAT GENERATION` — IMPLEMENTED. `*GAP CONDUCTANCE` (coefficient h_c) and `*GAP HEAT GENERATION` (generated flux q) parse onto the `*SURFACE INTERACTION` (`GapConductance{h,has}` / `GapHeatGeneration{q,has}`, core/contact.hpp), resolved by `Model::contact_conductance/heat_generation` and gated by `Model::has_thermal_contact()`. The thermal driver builds `fem::build_thermal_contact_pairs` (the SAME node-to-surface search as mechanical contact — task 1.6 shared interface geometry: slave nodes + master faces + grid + per-slave tributary area from the surface face integrals) and `fem::add_thermal_contact` adds, for each slave node in contact (projected gap ≤ adjust in the current config; the reference config for a pure thermal step), the conductance coupling `Q = h_c A (T_slave − Σ N_i T_master_i)` — the symmetric conductance block `h_c A B^T B` over the slave↔master temperature DOFs into the full thermal operator (`seed_thermal_contact` in numerics/heat_transfer.cpp, wired into both the steady and transient base builders) — plus the gap heat source `q A / 2` split between the surfaces. A thermal deck with a `*CONTACT PAIR` routes to `solve_heat_transfer` (procedure-first dispatch); a deck with no thermal-contact card is byte-for-byte the pre-contact thermal path. VALIDATED ANALYTICALLY (tests/test_thermal_contact.cpp + python test_thermal_contact_gap_conductance_python): two unit cubes stacked in z touching ONLY through a `*GAP CONDUCTANCE`, ends held at T_cold/T_hot — the three series resistors (cube 1/k, gap 1/h, cube 1/k) give the through-flux `Q = ΔT/(2/k + 1/h)` and the conductance-controlled interface drop `ΔT_gap = Q/h`, so the interface flux `q = h_c ΔT_gap = Q` to ~1e-6, and the cold-face RFL reaction balances Q (energy conservation); a stiffer h_c shrinks the interface drop toward perfect thermal contact; `*GAP HEAT GENERATION` deposits q·A at the interface with both ends held equal and the total RFL balances it (energy balance). No clean in-scope stock-CalculiX gap deck is gated (contact7/8/9 are transient coupled thermomechanical decks on C3D8I incompatible-mode elements, out of scope) — the analytical series-resistance references validate the physics. (spec: contact — thermal contact conductance and gap heat generation.)
- [x] 2.5 Contact modifiers + output (`*CLEARANCE`, `*CONTACT FILE`/`PRINT`/`OUTPUT`, CSTR) — IMPLEMENTED. CONTACT OUTPUT: `fem::recover_contact` re-runs the node-to-surface projection at the converged displacement and reports per-slave-node `ContactPoint{node_id, closed, p, gap, tau}` (status OPEN/CLOSED, normal pressure p = F_normal/tributary_area, signed gap, tangential friction traction) using the SAME penalty/friction laws as the operator; the nonlinear driver fills `StaticFields.contact` (empty on a contact-free deck). The writers emit CSTR: `.dat` gains a "contact stress (slave node, status, pressure, gap, tang.traction)" block and `.frd` a nodal CONTACT dataset (CPRESS/CSTATUS/CGAP); the Python result dict carries a `contact` list of per-node dicts. `*CONTACT FILE`/`*CONTACT PRINT`/`*CONTACT OUTPUT` are accepted (output-request cards, their CSTR/CDISP data lines ignored like *NODE PRINT). `*CLEARANCE, MASTER=, SLAVE=, VALUE=` sets a uniform initial clearance on the matching pair (`ContactPair::clearance`, order-independent surface match) that the operator ADDS to the geometric gap (`g_eff = g_geometric + clearance`) in both `add_contact` and `recover_contact`, so the interface starts at exactly VALUE regardless of the as-meshed geometry. VALIDATED (tests/test_contact.cpp): the two-block pressed-together solve reports every slave node CLOSED with positive pressure whose sum×area balances the applied load and a small penetration gap (CSTR recovery); a `*CLEARANCE` larger than a known penetration OPENS the interface (status flips to OPEN, gap = g_geo + clearance, zero pressure) while zero clearance leaves the geometric gap unchanged. (spec: contact — contact modifiers and output.)

- [x] 3.1 Steady-state conduction element kernels; `*HEAT TRANSFER, STEADY STATE` — scalar 1-DOF/node temperature field parallel to the mechanical path: `fem::element_conduction` (Kt_e = Σ_gp (g_a·g_b) k detJ w, reusing the shared shape functions/Gauss/`physical_gradients`), `fem::assemble_conduction` (thermal DOF numbering + Dirichlet elimination of prescribed temperatures), `*CFLUX`/`*DFLUX` into the thermal rhs (`fem::thermal_load_vector`; DFLUX reuses a generalized face integrator now covering tet/hex/wedge faces), and `numerics::solve_heat_transfer` (SPD Kt T = q via `solve_reduced`). Transient (3.2)/film (3.3)/radiation (3.4) remain deferred.
- [x] 3.2 Transient conduction (backward Euler) — capacitance `C_e = ∫ rho*c N_a N_b dV` (`fem::element_capacitance`, `Model::element_heat_capacity` from `*SPECIFIC HEAT` × `*DENSITY`) assembled on the full node numbering (`fem::assemble_full_thermal`) and time-stepped by `numerics::solve_transient`: backward Euler `(C/dt + Kt_film) T_{n+1} = (C/dt) T_n + q`, marching the step period (`increment.total`) in `increment.initial` sub-steps. VALIDATED analytically: (a) a unit cube convecting on all six faces keeps the field spatially UNIFORM, so the FEM reduces EXACTLY to the lumped ODE `Cv dT/dt = -h A (T-T_sink)` — the driver reproduces the backward-Euler recurrence to ~1e-8 (accounting for the default linear ramp of the unamplified sink); (b) a long transient relaxes to the independently-computed steady conduction+film field (~1e-3). (tests/test_thermal_transient.cpp)
- [x] 3.3 Convective film loads (`*FILM`) — `q_film = h (T - T_sink)` on a face adds `h ∫N_a N_b dA` to `Kt` and `h T_sink ∫N_a dA` to the rhs, via the shared face surface integrator `fem::face_surface_integrals` (reuses the `*DFLUX`/pressure face quadrature, all tet/hex/wedge faces). Wired into both the steady (`assemble_conduction` / `assemble_full_thermal`) and transient paths; parser handles `*FILM elem,F<face>,sink,h`. VALIDATED against stock CalculiX `oneel20fi` (single C3D20R cube, `*FILM 1,F1,-12,10`): nodal temperatures (NT −2/−1/0 profile) and heat generation (RFL, conduction-only) match to ~1e-14 (Python regression corpus). Forced-convection NETWORK film coupling is out of scope (Phase-5 1-D networks) — solid-only surface film done.
- [x] 3.4 Radiation (`*RADIATE`) — SURFACE-TO-AMBIENT `q = eps sigma (T^4 - T_amb^4)` implemented and linearized (Newton on `dq/dT = eps sigma 4 T_abs^3`) inside the thermal iteration, in absolute temperature via `*PHYSICAL CONSTANTS` (ABSOLUTE ZERO / STEFAN BOLTZMANN); works in both steady and transient. Parser handles `*RADIATE elem,R<face>,ambient,eps`. VALIDATED analytically against a conduction↔radiation equilibrium (`k(T_hot−T)/L = eps sigma (T_abs^4−T_amb_abs^4)`, scalar-root reference, ~1e-3). CAVITY radiation (`*RADIATE ...,CR`, gray-body surface-to-surface exchange) is now IMPLEMENTED: each `R<face>CR` face is a diffuse-gray enclosure patch (`Radiate::cavity`, parser strips the `CR` suffix). `fem::build_cavity` (src/fem/cavity_radiation.cpp) gathers each patch (centroid/area/outward-normal + shared face integrals, normal oriented out of the solid via the element centroid) and computes the view-factor matrix `F_ij = (1/A_i) ∫_Ai ∫_Aj cos_i cos_j/(pi r^2) dA_j dA_i` by direct double-area quadrature over a REFINED (sub=6) face sampling (`fem::face_gauss_points(...,sub)`, reusing the `*FILM`/`*DFLUX` face machinery), then row-normalizes to the summation rule `Σ_j F_ij = 1` (closed enclosure). `fem::cavity_heat_flow` solves the gray-body radiosity system `(I-(1-eps)F) J = eps sigma T^4` (dense Gauss-Jordan; black-body `eps=1` handled directly) and returns the per-patch net heat loss `Q_i` PLUS its exact temperature tangent `dQ_i/dT_k`; the driver (`numerics::add_cavity_radiation`) distributes `Q_i/A_i` over the patch nodes via the face load shape and assembles the full patch-coupled tangent into the same Newton system as surface-to-ambient radiation (steady + transient share the one nonlinear loop; the cavity geometry is built once per step). VALIDATED ANALYTICALLY (tests/test_cavity_radiation.cpp): (a) the summation rule `Σ_j F_ij = 1` and, for two facing patches, `F12=F21=1`; (b) the raw double-area kernel reproduces the KNOWN cube view factors (hollow-cube cavity: opposite face F≈0.1998, adjacent F≈0.2001, Hottel) within ~2e-2 on the coarse quadrature; (c) two facing patches (a 2-body enclosure) give the classic gray-body parallel-plate net flux `q = sigma (T1^4−T2^4)/(1/e1+1/e2−1)` to ~1e-9 across several emissivity pairs incl. the black-body limit, with the analytic tangent finite-difference-checked; PLUS a Python end-to-end deck (python test_cavity_radiation_parallel_plates_python) where two thin high-k plates held at 800/500 K exchange only by cavity radiation and the hot-plate flux reaction equals the parallel-plate `q` (≈13120 W) with energy conserved. No clean in-scope stock-CalculiX cavity `.dat.ref` is matched: the stock decks (beamhtcr/axrad) rely on CalculiX's hemicube+shadowing view-factor engine, out of scope for the direct-quadrature method — the analytical parallel-plate/cube references validate the physics instead. The `*VIEWFACTOR` read/write-to-file control (a solver-workflow directive, not the physics) remains rejected with an actionable message. (spec: heat-transfer-analysis — radiation with view factors.)

## 4. Coupled thermomechanics (spec: heat-transfer-analysis — coupled)

- [x] 4.1 `*COUPLED TEMPERATURE-DISPLACEMENT` — MONOLITHIC 4-DOF/node (u,v,w,T) coupling implemented and validated (and the one-way path retained as the decoupled limit). The parser recognizes `*COUPLED TEMPERATURE-DISPLACEMENT[, SOLUTIONS=MONOLITHIC|STAGGERED]` (`Procedure::Coupled` + `Model::coupled_scheme`), routing the thermal cards (temperature BCs on DOF 11, `*CFLUX`/`*DFLUX`/`*FILM`) to the scalar field and the mechanical loads to the displacement field. `numerics::solve_coupled` dispatches on the scheme. The MONOLITHIC path (`numerics::solve_monolithic_once`, src/numerics/heat_transfer.cpp) assembles ONE combined reduced system that stacks the mechanical free block `[0,nf_m)` and the thermal free block `[nf_m,nf_m+nf_t)` — `[[K_uu, K_uT],[0, K_TT]] [u;T] = [f_ext; q]` — reusing the already-reduced mechanical (`assemble_linear_static`, with its constraint transform) and thermal (`assemble_conduction`, with its Dirichlet map) operators, and the NEW off-diagonal `K_uT` thermal-strain coupling block `C_e[a][j] = ∫ B_aᵀ D {alpha,alpha,alpha,0,0,0} N_j dV` (`fem::element_thermal_coupling`, the exact derivative of the thermal-strain load w.r.t. nodal temperature), reduced through the mechanical transform (rows) and the thermal Dirichlet map (cols) with prescribed temperatures / the `-Tref` constant moved to the rhs, then solved once via `solve_reduced`. The mechanical->thermal block `K_Tu` is the plastic-dissipation heat source (see below); with no two-way term the 4-DOF system is block-triangular, so the monolithic solve reproduces the sequential thermal-then-mechanical result EXACTLY. CLI + Python (`solve`/`solve_text`) auto-dispatch on the procedure and the result dict carries both temperature and displacement/stress/reaction. VALIDATED: (a) analytical thermal stress end to end — a bar with a prescribed linear temperature profile and u_x fixed on both end faces develops sigma_xx = -E alpha T_bar (tests/test_thermal_stress.cpp test_coupled_deck_end_to_end + python test_coupled_temperature_displacement_thermal_stress); (b) MONOLITHIC == STAGGERED (== one-way) for the thermal-stress deck to solver tolerance (test_monolithic_equals_oneway + python test_coupled_monolithic_equals_staggered, temperature ~1e-9, stress ~1e-6); (c) the genuine two-way term — plastic-dissipation (Taylor-Quinney) heating — makes both schemes agree on the joint (T,u) state (test_two_way_plastic_dissipation).
- [x] 4.2 Iterated (Gauss-Seidel) STAGGERED coupling — implemented with an outer convergence check (`numerics::solve_coupled_staggered`). It alternates a thermal solve (`solve_heat_transfer`) and a mechanical solve (`solve_linear_static`, or `solve_nonlinear_static` for a plastic model), feeding the mechanical plastic-dissipation heat back into the next thermal solve, and exits when the temperature field stops changing (`< 1e-9`). The mechanical->thermal two-way term is plastic-dissipation heating: a Taylor-Quinney fraction `Model::taylor_quinney` of the per-element plastic work density `sigma_y(ep) ep V_e` (`Model::plastic_dissipation_heat`, from the per-element committed eqplastic that `solve_nonlinear_static` now reports via its new `eqplastic_by_elem` out-parameter) is deposited as a lumped nodal heat source (`*CFLUX`-equivalent, energy-conserving). CONVERGES IN ONE PASS when the coupling is one-way (`taylor_quinney==0` or no plasticity) — byte-for-byte the historical sequential result — and ITERATES when the two-way term is active. VALIDATED: test_two_way_plastic_dissipation (a stretched-past-yield element deposits dissipation heat; the staggered outer count > 1 and the temperature rises above the conducted baseline, while monolithic converges to the same state) + test_monolithic_equals_oneway (outer count == 1 for the one-way case). `numerics::solve_coupled_reported` exposes the outer-iteration count.
- [x] 4.3 Thermal expansion coupling verified against reference thermal-stress decks — `*EXPANSION[, ZERO=Tref]` parsed into `Material::expansion` (isotropic alpha + reference/stress-free temperature Tref) and consumed by the mechanical path via `Model::element_expansion()`. The equivalent thermal-strain load f_th = ∫ Bᵀ D (alpha (T - Tref)) dV (`fem::element_thermal_load` / `fem::thermal_strain_load_vector`, reusing the shared shape functions / Gauss / physical_gradients) is added to the mechanical rhs, and stress recovery uses sigma = D (eps_mech - eps_th), so a temperature field induces the correct thermal stress/strain while the internal force f_int = Ke u - f_th keeps the reactions RF consistent.
