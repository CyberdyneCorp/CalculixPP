# Phase 2 — Nonlinear Statics & Materials

Implementation against the front-loaded baseline plus the one new capability
(`nonlinear-solution-control`) added by this change. Each task cites the capability it
satisfies. Depends on `phase-1-foundation`.

## 1. Nonlinear solution driver (spec: nonlinear-solution-control [NEW]; static-analysis — "Nonlinear static analysis", "Incrementation control")

- [x] 1.1 `NonlinearSolver` Newton-Raphson loop: tangent assembly → residual → solve → update (`numerics::solve_nonlinear_static`; tangent = `assemble_linear_static` reduced system, residual via extracted `fem::internal_force`, solve via `solve_reduced`/`resolve_solver_kind`)
- [x] 1.2 Residual/convergence controls (force + displacement) configurable via `*CONTROLS` (`NonlinearControls`, documented defaults; parser reads `PARAMETERS=FIELD` and `TIME INCREMENTATION`)
- [x] 1.3 Automatic incrementation engine with cutback on divergence; `DIRECT` fixed increment; `*TIME POINTS` (`Incrementation`, `run_increments`/`clamp_increment`; grows after easy convergence, halves on failure, aborts below min)
- [x] 1.4 Line search (optional, behind a flag) for difficult increments (`NonlinearOptions::line_search`, OFF by default; scales the Newton update to reduce the residual)
- [x] 1.5 Validation: a linear deck solved in one increment reproduces the Phase-1 direct solve exactly (single-tet + beam10p reproduce `solve_linear_static` to rel-L2 < 1e-10; `tests/test_nonlinear.cpp`, `python/tests/test_regression.py`)
- [~] 1.6 `*STATIC, PERTURBATION` about a preloaded base state (stress stiffening) — DEFERRED: the geometric stress-stiffening K_geo needs geometric nonlinearity (NLGEOM) that does not exist yet; only linear elasticity is implemented, so a faithful perturbation-about-preload solve cannot be built without faking K_geo. Revisit with workstream 3/4 (finite-strain elements + plasticity).

## 2. Amplitude & load breadth (spec: loads-and-boundary-conditions)

- [x] 2.1 Amplitude engine: step, tabular, and periodic `*AMPLITUDE`, sampled per increment (`core/amplitude.hpp` `Amplitude::value_at`; `Model::amplitude_factor`; `*CLOAD`/`*DLOAD`/`*BOUNDARY` carry `AMPLITUDE=`; driver samples per-increment via `fem::external_load_vector(model, lambda)` and amplitude-scaled prescribed BCs; `tests/test_loads.cpp`)
- [x] 2.2 Body loads: gravity and centrifugal `*DLOAD`; `*DSLOAD` (`BodyLoad` GRAV/CENTRIF integrated as `rho*N` over element volume via the shape/Gauss rule; `*DSLOAD` reuses the pressure-face machinery; `fem::external_load_vector` extended; gravity global-equilibrium + centrifugal outward-radial tests in `tests/test_loads.cpp`)
- [~] 2.3 Step/restart property changes: `*CHANGE MATERIAL`, `*CHANGE PLASTIC`, `*CHANGE SOLID SECTION`. `*CHANGE SOLID SECTION` (MATERIAL=/ELSET=) is DONE — parsed and applied by appending a `SolidSection` that wins per element (`element_elastic` is last-writer); re-binds material within the single step. `*CHANGE MATERIAL, NAME=` is parsed (re-opens the named material, rejects unknown names) and `*CHANGE PLASTIC` data is accepted, but PARTIAL/DEFERRED: it only ever changes plastic hardening data, and plasticity does not exist yet (workstream 4), so the change is a no-op. The *cross-step / restart* aspect (redefining properties at a step boundary) also needs multi-step step handling, which the single-step model lacks. Parser + within-step apply for solid-section rebind implemented and tested (`tests/test_step_changes.cpp`).
- [~] 2.4 Load accumulation semantics (`OP=MOD` / `OP=NEW`). Parsed and applied *within the single step*: `OP=` on `*CLOAD`/`*DLOAD`/`*BOUNDARY` is validated (MOD default / NEW), and `OP=NEW` resets the accumulated loads of that type (concentrated / distributed+body / SPC), honoring the "first card of the type only" rule via per-type once-per-step flags. DEFERRED: true *across-steps* semantics — OP=MOD keeping prior-step loads while adding same-step same-DOF forces, and OP=NEW removing prior-step loads — require multi-step step handling that does not exist yet (the model is single-step; there is no step boundary to accumulate across). Within-step reset implemented and tested (`tests/test_step_changes.cpp`).

## 3. Element library expansion (spec: element-sections, mesh-and-model)

- [x] 3.1 Shared isoparametric kernel parameterized by topology (shape fns, Jacobian, integration) — material-point interface added. `fem/material_model.hpp` defines `MaterialModel` (`evaluate(strain, State) -> {stress, tangent}`), `ElasticIsoMaterial` (D from `elastic_iso_D`, sigma=D*strain), and a per-Gauss-point `MaterialState` (empty for elastic). `element_tangent_force` assembles `K_e = Σ_gp Bᵀ D_t B detJ w` and `f_e = Σ_gp Bᵀ σ detJ w`, evaluating the material at each Gauss point from the current element strain; shape/Jacobian/Gauss access is topology-parameterized (foundation for hex/wedge). `element_stiffness` delegates to the shared per-GP accumulation so C3D4/C3D10 results are bit-identical. `assemble_material_tangent` + `make_material_points` (fem/assembly) reassemble the reduced tangent + internal force from the material-point kernel; `solve_nonlinear_static` now reassembles both each Newton iteration at the current displacement + per-point state. Validated: material-point path reproduces `element_stiffness` to ~1e-12 and gives `f_e = Ke·ue` (`tests/test_element.cpp`); beam10p rel-L2 unchanged at 5.363e-8; nonlinear-reproduces-linear (single-tet + beam10p) still < 1e-10. NLGEOM/hex families deferred to 3.2.
- [x] 3.2 Hex/wedge families: `C3D8`, `C3D20`, `C3D6`, `C3D15` full integration (+ `C3D8R`/`C3D20R` reduced, see 3.3). Standard isoparametric shape functions and Gauss rules on the shared material-point kernel: `C3D8` trilinear (2x2x2), `C3D20` serendipity (3x3x3), `C3D6` linear wedge (1-tri x 2-Gauss), `C3D15` quadratic wedge (3-tri x 2-Gauss). Added to `ElementType`/`nodes_per_element`/`parse_element_type`/`element_type_name`; parser + mesh accept the TYPE= names and node counts; `frd_elem_type` extended for `.frd` output; nodal-stress extrapolation falls back to the Gauss-point mean for the new families (order-0 recovery — nodal stress is not on the displacement gate). Validated: partition-of-unity, exact Gauss-rule volume, constant-strain patch test (strain recovered to ~1e-11 at every Gauss point), 6-mode rigid-body null space, and material-point kernel reproduces `element_stiffness` with `fe = Ke·ue` for all families (`tests/test_element.cpp::test_hex_wedge`); end-to-end single-hex uniaxial patch (exact axial strain + Poisson contraction + reaction, `tests/test_hex.cpp`). Reference-deck validation: `beam8p` (C3D8) and `beam20p` (C3D20) reproduce stock CalculiX `.dat.ref` displacements to rel-L2 ~5.7e-8 (`python/tests/test_regression.py`). Pressure faces (*DLOAD P<face>) for hex/wedge deferred (face_nodes throws a clear error); body loads (GRAV/CENTRIF) already work for all topologies via the volume Gauss rule.
- [x] 3.3 Reduced-integration `C3D8R` (single centroid point) with Flanagan-Belytschko hourglass control (`build_hourglass`/`accumulate_hourglass` in `fem/element.cpp`): 4 corrected hourglass vectors γ_α orthogonal to the rigid-body + uniform-strain fields, stabilization modulus κ = 0.05·μ·V·Σ|b_j|² added to the tangent (and the resisting force K_hg·u to the internal force so the nonlinear residual stays consistent). `C3D20R` uses a genuinely reduced 2x2x2 rule (vs 3x3x3 full) which is rank-sufficient without hourglass control. Validated: C3D8R keeps exactly the 6 rigid-body zero-energy modes, passes the constant-strain patch test unchanged (γ orthogonal to linear fields), gives a strictly positive energy to a pure hourglass mode (spurious modes suppressed), and solves the end-to-end uniaxial patch identically to full C3D8 (`tests/test_element.cpp`, `tests/test_hex.cpp`). LIMITATION: hourglass control is implemented for C3D8R only; the 0.05 coefficient is a fixed documented default (not user-tunable via *HOURGLASS STIFFNESS yet); C3D8R pressure faces are not wired.
- [~] 3.4 Shell / beam / membrane sections (`*SHELL SECTION`, `*BEAM SECTION`, `*MEMBRANE SECTION`) — DEFERRED (not implemented). Full shell/beam formulations (drilling DOFs, through-thickness integration, offset/normal handling, the S8/B32 expansion CalculiX does) are a large workstream and cannot be done faithfully in this change without faking the kinematics. No parser/section support added; decks using these sections are rejected as before. Revisit as a dedicated structural-element workstream.
- [x] 3.5 Discrete/connector elements: `*SPRING` (stiffness contribution implemented), `*MASS`/`*DASHPOT` (properties stored for dynamics). New `core/connector.hpp` (`Spring`/`PointMass`/`Dashpot`) on the Model; parser accepts connector `*ELEMENT` TYPE= (`SPRINGA`/`SPRING1`/`SPRING2`, `MASS`, `DASHPOTA`/`DASHPOT1`/`DASHPOT2`) kept outside the isoparametric Mesh, and `*SPRING`/`*MASS`/`*DASHPOT` cards attach the property per elset. `assemble_linear_static`/`assemble_material_tangent` add the spring stiffness (SPRING1 grounded scalar k; SPRING2 [[k,-k],[-k,k]]; SPRINGA k·[[nnᵀ,-nnᵀ],[-nnᵀ,nnᵀ]] along the node line) via static condensation, with K_spring·u added to the nonlinear internal force. Masses/dashpots are stored and inert in a static solve (consumed by Phase-4 dynamics). Validated against the analytical single-spring u = F/k for all three spring kinds and mass/dashpot storage (`tests/test_connector.cpp`); matches the CalculiX spring1 setup (k=10, F=1 -> u=0.1). LIMITATION: linear springs only (NONLINEAR force-displacement tables not supported).
- [~] 3.6 `*USER ELEMENT` / `*USER SECTION` extensibility; `*NODAL THICKNESS` / `*NORMAL`; `*DISTRIBUTION` — DEFERRED (not implemented). User-element/section plug-in infrastructure and nodal thickness/normal/distribution fields are out of scope for this element-breadth change (thickness/normal are only meaningful once shells/beams from 3.4 exist). No parser support added.

## 4. Constitutive kernels (spec: material-models)

- [x] 4.1 Return-mapping rate-independent J2 (von Mises) plasticity: `*PLASTIC` (piecewise-linear yield/plastic-strain table) with isotropic, kinematic (linear back stress), and combined hardening, plus `*CYCLIC HARDENING`. New `J2PlasticMaterial` (`fem/material_model.hpp`, `src/fem/plasticity.cpp`) does a radial-return mapping; per-integration-point `MaterialState` carries plastic strain + equivalent plastic strain + back stress, split into COMMITTED (start-of-increment) and TRIAL (current Newton iteration) slots. `evaluate()` always return-maps from the committed history, so reassembling the tangent every Newton iteration is safe; the driver calls `MaterialPoints::commit()` only when an increment converges (`solve_nonlinear_static`). `make_material_points` builds `J2PlasticMaterial` where the element's material has `*PLASTIC`, else the elastic model — so elastic decks are byte-for-byte unchanged. Parser accepts `*PLASTIC[, HARDENING=ISOTROPIC|KINEMATIC|COMBINED]` and `*CYCLIC HARDENING` data (`Plastic` on `Material`); `Model::has_plasticity()` routes a plastic deck to `solve_nonlinear_static` (auto-dispatch in the bindings' `solve`), load applied incrementally. Plastic stress recovery reads the committed state (`recover_fields(model, fields, mp)`). The residual reference force was extended to use the reaction magnitude for displacement-controlled steps (||f_ext||==0). Validated (a): single-C3D8 uniaxial tension past yield reproduces the analytic hardening curve sigma = sy0 + H_iso*eps_p — displacement-controlled to rel 0 and load-controlled to rel ~1e-14 (`tests/test_plasticity.cpp`, `python/tests/test_regression.py`). Validated (b): NO clean in-scope `*PLASTIC *STATIC` C3D4/C3D10 `.dat.ref` deck exists in the reference corpus (all plasticity decks use NLGEOM, multi-step, orthotropic `*ELASTIC,TYPE=ORTHO`, or Johnson-Cook rate dependence — out of scope), so plasticity is validated against the analytic uniaxial solution as stated. GATES held: beam10p rel-L2 unchanged at 5.363e-8; nonlinear-reproduces-linear (single-tet + beam10p) still < 1e-10.
- [x] 4.2 Algorithmically CONSISTENT (algorithmic) tangent for J2 plasticity (Simo & Hughes Box 3.4), built in the engineering-shear Voigt convention as De minus the deviatoric-softening and n⊗n corrections; gives quadratic Newton convergence (uniaxial step converges in 2 iterations, |r| dropping 1e3 -> 1e-13 in one update). Finite-difference tangent verification test perturbs each of the 6 strain components about a plastic state and compares the numerical d sigma/d strain to the returned 6x6 tangent — agreement to rel ~5e-10 for both isotropic and combined hardening (`tests/test_plasticity.cpp::test_consistent_tangent_fd`).
- [~] 4.3 Hyperelasticity / foam: `*HYPERELASTIC`, `*HYPERFOAM` (with near-incompressible u/p path). PARTIAL — a compressible neo-Hookean law (`*HYPERELASTIC, N=1`, coefficients C10/D1) is implemented as a `HyperelasticNeoHookean` MaterialModel (`fem/material_model.hpp`, `src/fem/hyperelastic.cpp`) that plugs into the existing small-strain B^T sigma / B^T D_t B assembly. Parser accepts `*HYPERELASTIC[, N=1]` (mu=2*C10, kappa=2/D1 derived); a hyperelastic material routes through the nonlinear driver (`has_nonlinear_material()`), and `element_elastic()` derives an equivalent (E, nu) linear seed for the DOF-map assembly. The near-incompressible u/p concern is addressed by building the tangent from a decoupled deviatoric+volumetric split so a large kappa/mu stays well-scaled; the deck currently REJECTS D1=0 (true incompressible u/p mixed formulation not built). DEFERRED: `*HYPERFOAM` (Ogden foam) and large-strain finite kinematics (deformation gradient / push-forward) — the model is small-to-moderate-strain only. Validated ANALYTICALLY: in the small-strain limit the tangent equals `elastic_iso_D(E, nu)` and an end-to-end uniaxial cube matches the linear-elastic solution to ~1e-9 (`tests/test_material_models.cpp`). No clean in-scope small-strain `*HYPERELASTIC` `.dat.ref` deck exists (reference hyperelastic decks use NLGEOM finite strain), so analytic validation is used as stated.
- [~] 4.4 Creep / visco: `*CREEP`, `*VISCO`, `*VALUES AT INFINITY` — DEFERRED (not implemented). Time-dependent constitutive integration needs a real-time increment (dt) threaded into the material-point `evaluate()` and a viscoelastic history in `MaterialState`; the current small-strain driver is quasi-static (pseudo-time load factor, no physical dt at the material point). A faithful creep/visco law cannot be built without that time plumbing, so no parser/model support was added. Revisit alongside a transient driver.
- [~] 4.5 `*DEFORMATION PLASTICITY`, `*MOHR COULOMB` + `*MOHR COULOMB HARDENING`, `*DAMAGE INITIATION` — DEFERRED (not implemented). Deformation plasticity (Ramberg-Osgood total-strain) and pressure-dependent Mohr-Coulomb yield are additional return-mapping kernels; damage initiation needs a criterion-tracking state flag. These are feasible on the existing MaterialModel seam but were not reached in this change (4.6 was prioritized and fully delivered, 4.3 partially). No parser/model support added; decks using these cards are rejected as before.
- [x] 4.6 User material: `*USER MATERIAL` C++20 interface, `*DEPVAR` state, `*RATEDEPENDENT` scaling. DONE — a full C++ plug-in path. `UserMaterialRegistry` + `register_user_material(name, factory)` (`fem/material_model.hpp`, `src/fem/user_material.cpp`) is the extensibility seam: a `*USER MATERIAL` card names a registered factory keyed by NAME= (or the material name), and `make_material_points` builds the `MaterialModel` from the parsed `UserMaterial` (constants, `*DEPVAR` count, `*RATEDEPENDENT` scale). `MaterialState` gained generic `depvar`/`committed_depvar` slots sized from `*DEPVAR`, committed on convergence exactly like the plastic history. Parser accepts `*USER MATERIAL[, CONSTANTS=n, NAME=]`, `*DEPVAR`, `*RATEDEPENDENT`; a user material routes through the nonlinear driver and stress recovery (`has_nonlinear_material()`), with `effective_elastic()` deriving an (E, nu) seed from the constants {E, nu} for the DOF-map assembly. A built-in reference law `LinearUserMaterial` (auto-registered as `LINEAR_ELASTIC_USER`) proves the path end to end. Validated (`tests/test_material_models.cpp`, `tests/test_parser.cpp`): the registry finds/builds the factory; the reference law reproduces `ElasticIsoMaterial` bit-close; `*RATEDEPENDENT` scales stress/tangent; `*DEPVAR` state is written by evaluate() and promoted by commit(); an end-to-end `*USER MATERIAL` cube solves IDENTICALLY to the same `*ELASTIC` cube (analytic linear-elastic solution) to ~1e-9; and a brand-new law registered FROM the test drives a full solve — proving arbitrary downstream user code plugs in without touching the solver core. GATES held: beam10p rel-L2 unchanged; nonlinear-reproduces-linear (single-tet + beam10p) still < 1e-10.

## 5. Constraints (spec: constraints)

- [x] 5.1 `*EQUATION` linear constraints eliminated at assembly (SPD-preserving transform). DONE — a `ConstraintTransform` (`fem/constraint_transform.{hpp,cpp}`) resolves each equation's first (dependent) term to a linear combination of master DOFs, then expands EVERY global DOF onto the free DOFs (SPCs + MPC slaves removed), following slave->master chains with cycle detection. Assembly (`build_dof_map` + `scatter_block`) applies the congruence transform Tᵀ K T so the reduced system stays symmetric/SPD and the existing sparse SPD solve path is reused unchanged in BOTH the linear (`solve_linear_static`) and nonlinear (`solve_nonlinear_static`) drivers. External loads and the residual/internal force are distributed through the same transform (`add_external_load`, `reduce_free`), and slave motion is reconstructed from masters at any load factor (`transform.displacement`, tracks amplitude-scaled SPCs). Parser reads `*EQUATION` (count line + node,dof,coeff triplets). VALIDATED against a stock-CalculiX `.dat.ref`: `achtel2` (C3D20R cube, 2-term equations tying node 178 to node 78 in all 3 DOFs) reproduces reference displacements to rel-L2 1.27e-7 and the eliminated DOFs equal their masters to <1e-10 (`python/tests/test_regression.py::test_equation_constraint_matches_calculix`); plus analytic two-node-move-together + 3-term-average tests (`tests/test_constraints.cpp`).
- [x] 5.2 `*MPC` PLANE / STRAIGHT / BEAM (and user-defined hook). DONE — `Mpc` card expanded into linear equations (`src/core/constraints.cpp`): BEAM ties all 3 translations of the dependent node to the second (small-displacement rigid link); STRAIGHT keeps governed nodes on the line through the first two base nodes (two transverse equations per node from the orthonormal frame); PLANE keeps governed nodes in the plane through the first three base nodes (normal-projection equation, rigid-plane base coupling). A `Kind::User` hook is parsed (unregistered -> no equations, documented). Parser: `*MPC` type + node/nset list on the data line. VALIDATED analytically: BEAM ties reproduce coincident motion (`tests/test_constraints.cpp::test_mpc_beam`); STRAIGHT and PLANE solve a loaded cube without singularity (verified end-to-end). LIMITATION: small-displacement linearization only (no finite-rotation MPC update); PLANE base-node coupling uses equal (1/3) weights (exact rigid-plane translation/rotation, centroid-consistent).
- [x] 5.3 `*RIGID BODY`. DONE — `RigidBody` card (NSET=/ELSET=, REF NODE=, optional ROT NODE=) expanded to MPCs: without a rotation node each set node's translation is tied to the reference (u_node = u_ref); with a rotation node the small-rotation kinematics u_node = u_ref + theta x (x_node - x_ref) are imposed via the reference translation + rotation-node DOFs. Parser + expansion in `src/core/constraints.cpp`. VALIDATED analytically: a top-face node set tied to a reference node moves identically to it under a reference load (`tests/test_constraints.cpp::test_rigid_body_translation`). LIMITATION: small rotation (linearized cross product); the rotation node needs 3 rotational DOFs which map onto its translational comps 1..3 in this structural (3-DOF/node) model.
- [x] 5.4 `*COUPLING` kinematic + distributing / `*DISTRIBUTING COUPLING`. DONE — `Coupling` card: KINEMATIC ties each surface node's selected DOFs rigidly to the reference node; DISTRIBUTING / `*DISTRIBUTING COUPLING` makes the reference DOF the (equal-weight) average of the surface nodes. Parser handles `*COUPLING, REF NODE=, SURFACE=` with a following `*KINEMATIC`/`*DISTRIBUTING` modifier (DOF-range data lines) and `*DISTRIBUTING COUPLING` (node/weight data lines); surfaces resolve to node lists (nset / *SURFACE NODE / *SURFACE ELEMENT face nodes). VALIDATED analytically: a kinematic coupling makes the surface nodes follow the reference node exactly (`tests/test_constraints.cpp::test_kinematic_coupling`). LIMITATION: distributing weights are equal (area/tributary weighting deferred); couplings act on translational DOFs.
- [x] 5.5 Mortar `*TIE` for matching + non-matching surfaces; over-constraint detection. Over-constraint detection DONE and tested: `build_constraint_transform` raises on a dependent DOF that is also a *BOUNDARY SPC, on a DOF that is the dependent term of more than one constraint, and on a cyclic slave->master dependency — each with the conflicting origins (`tests/test_constraints.cpp::test_overconstraint_slave_is_spc`, `test_overconstraint_double_dependent`). `*TIE` for MATCHING surfaces DONE: each slave node is paired with the coincident master node within POSITION TOLERANCE and all 3 translations tied; validated end-to-end by bonding two stacked C3D8 cubes at a shared interface (slave nodes follow masters to 0.0, load transfers through). NON-MATCHING (mortar) is PARTIAL/[~]: a slave with no master inside tolerance is skipped (nearest-node association only; a true mortar projection/integration is a larger workstream, deferred). GATES held: beam10p rel-L2 unchanged at 5.363e-8; nonlinear-reproduces-linear (single-tet + beam10p) still < 1e-10; a *EQUATION model reproduces its linear solve in the nonlinear driver to <1e-10.
